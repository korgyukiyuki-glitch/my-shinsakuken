# 競合分析: マイ診察券

> 作成日: 2026年3月8日
> 対象プロダクト: マイ診察券 v1.0（メディカルクラフトラボラトリー）

---

## 1. 市場概要

### 1.1 市場規模と成長率

| 指標 | 数値 | 出典 |
|------|------|------|
| 日本のmHealth市場（2024年） | 72億ドル（約1.1兆円） | IMARC Group |
| 日本のmHealth市場（2033年予測） | 514億ドル（約7.7兆円） | IMARC Group |
| CAGR（2025-2033） | 22.7% | IMARC Group |
| 日本の医療DX市場（2025年） | 7,818億円 | 富士キメラ総研 |
| ヘルスケア産業全体（2025年） | 約33兆円 | 経済産業省 |

### 1.2 TAM / SAM / SOM

| セグメント | 算出根拠 | 推定規模 |
|------------|----------|----------|
| **TAM** (Total Addressable Market) | 日本の通院患者数 × スマホ保有率。外来患者数は年間約7億人回（延べ）、実患者数は推定4,000万人以上 | 4,000万人 |
| **SAM** (Serviceable Available Market) | 複数医院に通院する患者。40歳以上の約60%が2つ以上の医院に通院 | 1,500万人 |
| **SOM** (Serviceable Obtainable Market) | 初年度にリーチ可能な層。ASO + 口コミでの自然獲得 | 5万〜10万人 |

### 1.3 主要顧客セグメント

1. **複数医院通院者（40-70代）** — 内科+歯科+眼科など3院以上に通院。診察券の管理が最も煩雑
2. **子育て世代（30-40代）** — 子供の歯科・小児科・耳鼻科の診察券を管理。家族分の一元管理ニーズ
3. **代理管理者（40-50代）** — 高齢の親の通院をサポート。遠方からの管理ニーズ
4. **定期通院者（20-30代）** — 歯科の定期検診など。デジタルネイティブ世代

### 1.4 市場の成功要因

- **導入障壁の低さ**: 患者がすぐ使い始められること（アカウント不要、医院導入不要）
- **日常的な利用習慣の形成**: 受付時に毎回使うことでリテンション確保
- **信頼性**: 医療情報を扱うためプライバシーとセキュリティへの配慮
- **全診療科対応**: 歯科だけでなく複数科をカバーする汎用性

### 1.5 市場動態

- **デジタル化の加速**: コロナ禍以降、非接触受付・オンライン予約の需要が急拡大
- **診察券アプリの乱立**: 2026年時点で12〜13種類の診察券アプリが存在（ITreview調べ）
- **全て医院主導型**: 既存アプリは全て医院側の導入が前提。患者主導型は空白地帯
- **医療機関数**: 病院8,060施設 + 一般診療所105,000施設以上 + 歯科68,000施設（2024年、厚生労働省）

## 2. 競合セット概要

### 2.1 ポジションマップ

```
            患者主導（医院導入不要）
                    ↑
                    │
                    │   ★ マイ診察券
                    │   （ブルーオーシャン）
                    │
    単一医院 ←──────┼──────→ 複数医院
                    │
         myDental   │  EPARKデジタル診察券
         e-Apo      │  M3デジスマ診察券
         Apotool    │
                    │
                    ↓
            医院主導（医院導入必須）
```

### 2.2 競合分類

| 分類 | 競合名 | 特徴 |
|------|--------|------|
| **マーケットリーダー** | 私の歯医者さん (Apotool) | 200万DL、歯科業界No.1 |
| **プラットフォーム型** | EPARKデジタル診察券 | 200万DL超、26,000医院連携 |
| **SaaS連動型** | M3デジスマ診察券 | エムスリー(東証プライム)、全診療科 |
| **ニッチプレイヤー** | myDental | ユーザー1万人超、歯科特化 |
| **ニッチプレイヤー** | e-Apo診察券 | ヨシダ(大手歯科メーカー)連携 |

### 2.3 隣接・間接競合

- **お薬手帳アプリ**: EPARKお薬手帳、e薬Link、日薬eお薬手帳 — 薬局側の機能で間接競合
- **健康管理アプリ**: Apple Health、Google Fit — 健康データの包括管理で間接競合
- **カレンダーアプリ**: Googleカレンダー、Appleカレンダー — 予約管理の代替手段
- **紙の診察券**: 最大の競合。変更コスト（スイッチングコスト）がゼロ

## 3. 競合プロファイル

### 3.1 私の歯医者さん (Apotool & Box)

| 項目 | 詳細 |
|------|------|
| **提供元** | 株式会社ストランザ |
| **リリース** | 2018年7月 |
| **累計DL** | 200万以上（2025年時点） |
| **対応診療科** | 歯科のみ |
| **導入モデル** | 医院主導型（Apotool&Box導入医院の患者のみ利用可） |

**主な機能**:
- QRコード受付
- アプリから予約・予約変更
- 家族の予約・治療履歴を一括管理
- キャッシュレス決済（ささっとPay）
- 術前・術後の注意事項メッセージ送信
- 呼出ポップアップ表示

**価格体系（医院側）**:
| 項目 | 金額 |
|------|------|
| Apotool基本プラン | 月額18,000円〜 |
| 診察券アプリ追加 | 初期330,000円 + 月額11,000円 |
| LINE連携 | 初期300,000円 + 月額25,000円 |
| Medical Box Note | 月額24,000円 |

**強み**:
- 歯科業界で圧倒的なブランド認知（DL数No.1）
- 機能の豊富さ（決済、メッセージ、カメラ機能）
- 開発から販売・サポートまで自社一貫体制
- IT導入補助金対象で導入ハードル低下

**弱み**:
- 歯科限定（内科・眼科等の患者は対象外）
- 医院がApotoolを導入しない限り患者は利用不可
- 医院の導入コストが高い（初期33万円+月額3万円〜）
- 単一医院の診察券しか保持できない

**脅威度**: ⚠️ **高**
- 最大リスク: 複数医院対応を実装した場合、200万DLのユーザーベースで一気にマイ診察券の差別化を侵食する可能性あり

### 3.2 EPARKデジタル診察券

| 項目 | 詳細 |
|------|------|
| **提供元** | エンパワーヘルスケア株式会社 |
| **累計DL** | 200万以上 |
| **対応診療科** | 全診療科（歯科・病院・クリニック） |
| **導入モデル** | プラットフォーム型（EPARK加盟施設のみ） |

**主な機能**:
- 複数医療機関の診察券をアプリ内管理
- 家族の診察券も一括管理
- ネット予約（26,000医院 + 24,000薬局）
- リマインド通知
- 受診履歴の記録（メモ・画像）
- 医療費管理 + e-Tax申告ファイル出力
- 院外順番待ち通知

**価格体系**: 患者側は無料。医院側はEPARKエコシステムへの加盟費用

**強み**:
- 全診療科対応（マイ診察券と同じ）
- 複数医療機関の一元管理（マイ診察券と同じ）
- 圧倒的なネットワーク規模（26,000医院）
- 医療費管理・e-Tax出力という独自機能
- 200万DL超の実績

**弱み**:
- EPARK加盟施設でないと診察券登録・予約ができない
- 患者が自分で自由に医院を追加できない
- EPARKアカウント登録が必須（プライバシー懸念）
- サーバーにデータ送信あり（ローカル保存ではない）
- UIが複雑（多機能ゆえの情報過多）

**脅威度**: ⚠️ **中〜高**
- マイ診察券と最も機能が近い競合。ただし「EPARK加盟医院限定」という制約が最大の差別化ポイント

## 5. 差別化の機会

### 5.1 既存競合が満たしていないニーズ

1. **「どの医院でもすぐ使える」体験**
   - 全競合が医院側のシステム導入を前提としている
   - マイ診察券は医院名と患者番号を入力するだけで即座に利用開始
   - QRリーダーがない医院でも、患者番号の大画面表示で受付可能

2. **プライバシーファースト設計**
   - 全競合がサーバーにデータを送信（アカウント登録必須）
   - マイ診察券は全データをデバイス内に完結（ゼロデータ収集）
   - 広告・トラッキングなし

3. **診療科を横断した統合管理**
   - 歯科のApotool、myDental、e-Apoは他科に対応不可
   - EPARKは全診療科だが加盟施設限定
   - マイ診察券は全ての医院を制限なく登録可能

4. **アカウント不要の即時利用**
   - 全競合がメールアドレスやアカウント登録を要求
   - マイ診察券はインストール → プロフィール（氏名のみ）→ 即利用開始

### 5.2 ポジショニングギャップ

マイ診察券が占める**ブルーオーシャン**:

```
┌─────────────────────────────────────────────┐
│         患者の利用開始に必要なもの            │
├──────────────┬──────────────────────────────┤
│ マイ診察券    │ スマホ + 患者番号（知っている）│
│ Apotool      │ 医院がApotool契約 + 患者登録  │
│ EPARK        │ EPARK加盟 + EPARKアカウント   │
│ M3デジスマ   │ 医院がデジスマ契約 + M3アカウント│
│ myDental     │ 医院がジニー契約 + 患者登録    │
│ e-Apo        │ 医院がヨシダ機器導入 + 患者登録 │
└──────────────┴──────────────────────────────┘
```

→ マイ診察券だけが「ゼロ・フリクション」で利用開始可能

## 6. ポジショニング戦略提言

### 6.1 推奨ポジショニング

**「患者が自分で使える、唯一のデジタル診察券」**

- 全競合は「医院→患者」のトップダウン型
- マイ診察券は「患者→医院」のボトムアップ型
- この構造的差異は競合が簡単には模倣できない（既存の医院向けSaaSビジネスモデルと矛盾するため）

### 6.2 強調すべき差別化要素

| 優先度 | 差別化要素 | メッセージング |
|--------|-----------|---------------|
| 1 | 医院導入不要 | 「今すぐ使える。医院への相談は不要です」 |
| 2 | 全診療科対応 | 「歯科も内科も眼科も。全ての診察券をひとつに」 |
| 3 | プライバシー | 「データは端末だけに。サーバー送信ゼロ」 |
| 4 | 完全無料 | 「全機能・枚数無制限、永久無料」 |

### 6.3 狙うべきセグメント

- **最優先**: 複数医院に通院する40-60代（3院以上が理想）
  - 理由: ペインが最も深い。紙の診察券を5枚以上持ち歩いている層
- **次優先**: 子供の通院管理をする30-40代の親
  - 理由: 小児科+歯科+耳鼻科で3枚以上。管理の煩雑さが明確

### 6.4 避けるべきセグメント

- **単一医院のみの患者**: Apotool等の方が医院連携で優れたUXを提供。マイ診察券の差別化が薄い
- **デジタルリテラシーが極端に低い高齢者**: サポートコストが高く、個人開発での対応が困難

### 6.5 12-18ヶ月の競合リスクと機会

#### リスク

| リスク | 影響度 | 発生確率 | 対策 |
|--------|--------|----------|------|
| **Apotoolが複数医院対応を実装** | 極めて高 | 中 | 先行者利益の確保（ユーザー基盤構築を急ぐ） |
| **EPARKが加盟施設外の診察券登録を許可** | 高 | 低 | プライバシー優位性とUXの差別化を維持 |
| **M3デジスマが患者直接利用を解放** | 高 | 低 | エムスリーの医院向けSaaSモデルと矛盾するため可能性は低い |
| **大手IT企業（LINE、Yahoo等）が参入** | 極めて高 | 低〜中 | ニッチ市場の深掘り、コアユーザーのロイヤリティ確保 |

#### 機会

| 機会 | インパクト | 実現可能性 |
|------|----------|------------|
| マイナ保険証の普及で「デジタル管理」への意識向上 | 高 | 高 |
