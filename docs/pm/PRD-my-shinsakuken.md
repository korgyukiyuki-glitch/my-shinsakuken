## 1. サマリー

おまとめ診察券は、複数の医院の診察券を1つのスマートフォンアプリで管理できる、患者主導型のデジタル診察券アプリである。歯科・内科・眼科・皮膚科など全ての診療科に対応し、医院側にシステム導入を一切求めない。患者は医院名と患者番号を手入力するだけで、すぐに使い始められる。データは全て端末内に保存され、サーバーへの送信やクラウド保存は行わない。

## 3. 背景

### 3.1 なぜ今このアプリが必要か

日本では、患者が複数の医院に通院することが一般的である。40歳以上の約60%が2つ以上の医院を持ち、高齢者では5院以上に通院するケースも珍しくない。各医院で発行される紙の診察券は財布やカードケースを圧迫し、紛失や忘れによる受付トラブルが日常的に発生している。

### 3.2 既存ソリューションの限界

2026年現在、診察券アプリは12〜13種類存在するが、**全てが医院主導型**である。医院がシステムを契約しなければ患者はアプリを使えず、かつ複数医院の横断管理もできない。患者目線での「診察券をまとめたい」というシンプルなニーズに応えるアプリが存在しない。

### 3.3 最近可能になったこと

- **スマホ普及率97%超**: 全世代でスマホが普及し、アプリベースのソリューションが現実的に
- **QRコード文化の定着**: コロナ禍以降、QRコードによる非接触受付が社会的に受容された
- **マイナ保険証の普及**: 医療分野のデジタル化に対する社会的認知が向上
- **個人開発の技術基盤成熟**: Expo / React Native によるクロスプラットフォーム開発が個人でも高品質なアプリを実現可能に

## 7. ソリューション

### 7.1 画面遷移とUX

全21画面構成。主要ユーザーフローは以下の通り:

**フロー1: 診察券登録**
```
ホーム → 「医院を追加」 → 医院名・患者番号入力 → カラー設定 → 登録完了
```

**フロー2: 受付で使う**
```
診察券一覧 → カードタップ → 詳細画面 → 患者番号大画面表示 or QRコード表示
```

**フロー3: 予約メモ**
```
予約一覧 → 「予約を追加」 → 医院・日時・内容入力 → 通知設定 → 登録完了
```

### 7.2 画面一覧

| カテゴリ | 画面 | 概要 |
